Data

See where the data for our machine learning model comes from and how we process it.

Last updated: October 19, 2022

Data sources

Our data comes from three sources; these are offical government statistics which demonstrates their accuracy and completeness.

  • Suburb and locality census data

    Australian Bureau of Statistics at https://dbr.abs.gov.au/index.html

    This dataset contains over 10,000 statistical attributes for every suburb and locality in Victoria.

  • Regional profiles

    Victorian Government at https://discover.data.vic.gov.au/dataset/2015-local-government-area-profiles

    This dataset contains many features about each Local Government Area in Victoria.

  • Crime statistics

    Victoria Police at https://www.crimestatistics.vic.gov.au/crime-statistics/download-crime-data

    This dataset contains characteristics of crime recorded by Victoria Police in the Law Enforcement Assistance Program (LEAP) for each suburb and locality.

Data processing

We used standard statistical analysis techniques with Python and the Pandas library to prepare our data for developing our machine learning model.

This included removing much of the unnecessary data; with over 10,000 features for each suburb, many of the features were unrelated to the goal of this application.

Machine Learning

Our machine learning model makes use of a generated dataset. This was necessary due to the lack of existing data about people's preferred suburb in Victoria based on their accomodation preferences.

With this dataset, we trained a Decision Tree model which gets used with your inputted preferences to recommend to you your ideal suburb.
